# AlphaPredator 阶段清单

## Phase 1：市场数据底座收尾

### 需要做什么
- 落地本地真实市场数据入口，不再主要依赖 sample data
- 建立股票池、日线行情、基础指标的本地导入链路
- 明确并落地 SQLite / DuckDB / Parquet 的职责分层
- 让市场总览页读取真实本地数据
- 让单股详情页读取真实本地数据
- 为后续 AI 结果、重点关注、历史记录预留稳定数据模型

### 达成目标
- 能导入并持久化本地市场数据
- 首页和详情页看到的不是示例数据，而是真实本地数据
- 后续热点板块、skill、AI 选股都能建立在这层数据底座上

---

## Phase 2：热点板块图片解析链路

### 需要做什么
- 按固定格式导入复盘图片
- 实现 `MMDD.png + 批次年份 -> trade_date` 的日期解析
- 完成 OCR 与个股行提取
- 保存 `reason_raw` / `reason_clean`
- 完成个股到板块的归一化映射
- 生成日度热点板块结果
- 生成近 3 个交易日派生结果
- 为低置信度结果打上 `needs_review`

### 达成目标
- 同格式图片能稳定解析
- 热点板块结果可持久化、可回溯
- 市场总览页可以消费“当日热点板块”和“近 3 日持续性/轮动”结果

---

## Phase 3：自然语言到 skill 的工作流

### 需要做什么
- 建立自然语言输入入口
- 让 AI 生成结构化 skill 草稿
- 提供 skill 结构化展示
- 支持 skill diff / 微调
- 强制手动确认后才能执行
- 保存 skill 版本历史
- 生成 change summary

### 达成目标
- 用户可以输入自然语言生成 skill
- skill 可以被确认、微调、回看
- AI 不能跳过确认直接执行筛选

---

## Phase 4：AI 两轮选股主链路

### 需要做什么
- 将 skill 编译为可执行筛选条件
- 建立确定性预筛选
- 生成候选股票特征包
- 完成模型池配置
- 实现第一轮独立选股
- 实现第二轮交叉互评
- 完成评分与投票收敛
- 让 AI 选股结果页接入真实结果
- 让单股详情页接入 AI 完整分析
- 保存当次 skill、模型参与情况、结论与建议等历史记录

### 达成目标
- 已确认 skill 可以真实执行
- 能得到最终候选股票列表
- 结果页不再是占位页，而是承接真实 AI 输出
- 单股详情页能看到 AI 结论、理由、风险和动作建议

---

## Phase 5：重点关注 / 重分析 / 偏好反馈 / 历史闭环

### 需要做什么
- 支持标记 / 取消重点关注
- 让重点关注页接入真实数据
- 支持对重点关注股票重新分析
- 提供动作变化 / 理由变化提醒
- 收集股票结果反馈
- 收集 skill 质量反馈
- 让偏好学习影响排序与结果解释
- 让历史记录页完整接入：按日期 / 按 skill / 按股票查看

### 达成目标
- 可以长期跟踪重点关注股票
- 可以保留并比较多次重新分析结果
- 反馈可以影响后续排序或模型权重
- 历史记录形成完整闭环
