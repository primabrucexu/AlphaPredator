# AlphaPredator 阶段清单

## 文档用途

- 本文件只维护 **阶段目标、阶段范围、关键任务、达成标准**，供 AI agent 和开发者理解“接下来要做什么”。
- `phase.md` 只描述计划，不描述当前实际完成度。
- 当前真实落地状态、是否完成、下一步推进情况统一记录在 `progress.md`。

## 维护规则

- 每个阶段都按固定模板维护，字段顺序保持一致。
- 只写该阶段的目标和计划，不写“已完成 / 未完成 / 进行中”。
- 如果阶段范围变化，优先更新：`本阶段范围`、`关键任务`、`达成标准`。
- 如果阶段之间存在依赖关系，写在 `前置依赖`，不要混写到 `关键任务` 中。

## 固定模板

以下每个阶段都按同一模板维护：

- 阶段目标：本阶段最终要实现什么
- 本阶段范围：本阶段明确覆盖哪些内容
- 关键任务：为达成目标必须完成的任务
- 达成标准：如何判断该阶段可以结束
- 前置依赖：进入本阶段前必须具备什么
- 非目标：本阶段明确不处理什么

## Phase 1：市场数据底座收尾

- 阶段目标：建立稳定的本地市场数据底座，让前端页面优先读取真实本地数据而不是示例数据。
- 本阶段范围：
  - 本地真实市场数据导入入口
  - 股票池、日线行情、基础指标导入链路
  - `SQLite / DuckDB / Parquet` 职责分层
  - 市场总览页与单股详情页的数据切换
  - 为后续 AI 结果、重点关注、历史记录预留基础数据模型
- 关键任务：
  - 落地本地真实市场数据入口，不再主要依赖 sample data
  - 建立股票池、日线行情、基础指标的本地导入链路
  - 明确并落地 `SQLite / DuckDB / Parquet` 的职责分层
  - 让市场总览页读取真实本地数据
  - 让单股详情页读取真实本地数据
  - 为后续 AI 结果、重点关注、历史记录预留稳定数据模型
- 达成标准：
  - 能导入并持久化本地市场数据
  - 首页和详情页看到的不是示例数据，而是真实本地数据
  - 后续热点板块、skill、AI 选股都能建立在这层数据底座上
- 前置依赖：
  - 前后端基础工程可运行
  - 本地数据目录与数据库目录已具备基础结构
- 非目标：
  - 不在本阶段实现 AI 两轮选股与历史闭环
  - 不在本阶段完成全市场实时/增量数据抓取能力

---

## Phase 2：热点板块图片解析链路

- 阶段目标：把固定格式复盘图片转成可落库、可回溯、可在页面消费的热点板块数据。
- 本阶段范围：
  - 图片导入规则与批次管理
  - `MMDD.png + 批次年份 -> trade_date` 日期解析
  - OCR、板块头识别、个股行提取
  - 原始原因与清洗原因保留
  - 个股到板块的归一化映射
  - 日度热点板块结果与近 3 个交易日派生结果
  - 低置信度结果的 `needs_review` 标记
- 关键任务：
  - 按固定格式导入复盘图片
  - 实现 `MMDD.png + 批次年份 -> trade_date` 的日期解析
  - 完成 OCR 与个股行提取
  - 保存 `reason_raw` / `reason_clean`
  - 完成个股到板块的归一化映射
  - 生成日度热点板块结果
  - 生成近 3 个交易日派生结果
  - 为低置信度结果打上 `needs_review`
- 达成标准：
  - 同格式图片能稳定解析
  - 热点板块结果可持久化、可回溯
  - 市场总览页可以消费“当日热点板块”和“近 3 日持续性/轮动”结果
- 前置依赖：
  - `Phase 1` 的本地数据底座已可用
  - 市场总览页已具备消费本地导入数据的主路径
- 非目标：
  - 不在本阶段解决全市场行情抓取与增量更新
  - 不在本阶段实现 AI 选股逻辑

---

## Phase 2.9：全市场数据接入与增量更新

- 阶段目标：接入全市场真实数据源，支持初始化入库、初始化进度查看与当日增量更新。
- 本阶段范围：
  - 市场数据源接入
  - 全市场交易数据持久化入库
  - 数据初始化页面
  - 初始化进度返回接口或机制
  - 当天市场交易数据增量更新链路
- 关键任务：
  - 接入可查询市场交易数据的市场数据源
  - 将交易数据持久化保存到数据库中，供后续分析使用
  - 设计并实现网页中的数据初始化页面
  - 提供市场初始化进度返回能力
  - 支持增量更新当天市场交易数据
- 达成标准：
  - 可以完成一轮全市场数据初始化入库
  - 网页中可以看到初始化进度
  - 支持在已有数据基础上执行当天增量更新
  - 初始化和增量更新结果可以被现有页面消费
- 前置依赖：
  - `Phase 1` 数据底座已可用
  - `Phase 2` 热点链路已稳定，不阻塞市场数据主链路扩展
- 非目标：
  - 不在本阶段实现自然语言生成 skill
  - 不在本阶段完成 AI 两轮选股结果页

---

## Phase 2.10：首页极简化与股票直达搜索

- 阶段目标：将首页调整为极简搜索入口，只保留 AlphaPredator 品牌与股票搜索框，并支持输入代码/拼音简称后直达详情页。
- 本阶段范围：
  - 首页布局改造为“品牌 + 搜索框”
  - 支持股票代码搜索
  - 支持拼音简称搜索
  - 点击搜索后直接跳转单股详情页
  - 搜索异常提示（空输入、无匹配、多匹配、服务异常）
  - 首页联动初始化就绪状态（Token / 股票清单 / 初始化状态）
  - 首页提供初始化入口跳转（未就绪时可快速进入初始化页）
  - 首次进入首页时展示“是否已完成市场数据初始化”状态提示
  - 初始化板块在 CSV 上传成功后隐藏上传模块，仅展示股票清单摘要信息
  - 首页展示股票清单更新时间与每日行情更新截止时间
- 关键任务：
  - 精简首页内容，移除市场总览、热点板块、股票列表等信息区块
  - 前端新增/改造搜索提交流程
  - 后端提供代码/拼音简称解析能力（返回唯一 `stock_code`）
  - 接通前端搜索与后端解析接口
  - 在匹配成功时跳转 `/stocks/{stock_code}`
  - 接入初始化就绪检查接口（Token 已配置、股票清单已上传、初始化状态可查询）
  - 未满足初始化前置条件时给出明确提示，并支持一键跳转初始化页面
  - 实现初始化板块的“上传后状态切换”：已上传时不再展示上传控件，改展示主板/创业板/科创板股票数量和最后更新时间
  - 接入首页时间信息展示：`stock_list_updated_at`、`daily_quote_cutoff_time`
- 达成标准：
  - 访问首页时仅展示 AlphaPredator logo/标题和一个搜索框
  - 输入股票代码可直接跳转到详情页
  - 输入拼音简称可解析并跳转到详情页
  - 异常输入可得到明确提示，不阻断页面使用
  - 初始化未完成或前置条件缺失时，首页可明确提示原因并提供初始化入口
  - 首次进入首页时可明确看到“是否已完成市场数据初始化”
  - CSV 上传成功后，初始化板块仅显示股票清单摘要（按市场板块数量 + 最后更新时间），不再显示上传模块
  - 首页可展示股票清单更新时间与每日行情更新截止时间
- 前置依赖：
  - 股票清单 CSV 已上传并落库（包含拼音简称字段）
  - 后端市场数据服务可查询股票基础信息
  - 初始化接口已支持 Token 配置、股票清单上传与状态查询
- 非目标：
  - 不在本阶段实现复杂筛选器、多条件联动搜索
  - 不在本阶段恢复首页市场总览看板

## Phase 3：自然语言到 skill 的工作流

- 阶段目标：建立从自然语言到结构化 skill 草稿，再到人工确认执行的完整工作流。
- 本阶段范围：
  - 自然语言输入入口
  - AI 生成结构化 skill 草稿
  - skill 结构化展示
  - skill diff 与微调
  - 强制人工确认后执行
  - skill 版本历史与 change summary
- 关键任务：
  - 建立自然语言输入入口
  - 让 AI 生成结构化 skill 草稿
  - 提供 skill 结构化展示
  - 支持 skill diff / 微调
  - 强制手动确认后才能执行
  - 保存 skill 版本历史
  - 生成 change summary
- 达成标准：
  - 用户可以输入自然语言生成 skill
  - skill 可以被确认、微调、回看
  - AI 不能跳过确认直接执行筛选
- 前置依赖：
  - `Phase 2.9` 已提供足够稳定的市场数据来源，供后续 skill 执行使用
  - 已有 AI 选股与 skill 相关设计文档
- 非目标：
  - 不在本阶段完成 AI 两轮选股打分与结果收敛
  - 不在本阶段实现重点关注与历史闭环

---

## Phase 4：AI 两轮选股主链路

- 阶段目标：把已确认的 skill 编译执行，完成预筛选、两轮 AI 评估、结果收敛与真实展示。
- 本阶段范围：
  - skill 编译为可执行筛选条件
  - 确定性预筛选
  - 候选股票特征包生成
  - 模型池配置
  - 第一轮独立选股
  - 第二轮交叉互评
  - 评分与投票收敛
  - AI 结果页与单股详情页真实接入
  - 历史记录保存当次 skill、模型参与情况、结论与建议
- 关键任务：
  - 将 skill 编译为可执行筛选条件
  - 建立确定性预筛选
  - 生成候选股票特征包
  - 完成模型池配置
  - 实现第一轮独立选股
  - 实现第二轮交叉互评
  - 完成评分与投票收敛
  - 让 AI 选股结果页接入真实结果
  - 让单股详情页接入 AI 完整分析
  - 保存当次 skill、模型参与情况、结论与建议等历史记录
- 达成标准：
  - 已确认 skill 可以真实执行
  - 能得到最终候选股票列表
  - 结果页不再是占位页，而是承接真实 AI 输出
  - 单股详情页能看到 AI 结论、理由、风险和动作建议
- 前置依赖：
  - `Phase 2.9` 已提供稳定的全市场数据来源
  - `Phase 3` 已提供已确认 skill 的输入
- 非目标：
  - 不在本阶段实现长期重点关注反馈学习闭环
  - 不在本阶段完成所有偏好学习机制

---

## Phase 5：重点关注 / 重分析 / 偏好反馈 / 历史闭环

- 阶段目标：形成围绕重点关注股票、反馈学习与历史记录的长期使用闭环。
- 本阶段范围：
  - 标记 / 取消重点关注
  - 重点关注页真实数据接入
  - 重点关注股票重新分析
  - 动作变化 / 理由变化提醒
  - 股票结果反馈与 skill 质量反馈
  - 偏好学习影响排序与结果解释
  - 历史记录按日期 / 按 skill / 按股票查看
- 关键任务：
  - 支持标记 / 取消重点关注
  - 让重点关注页接入真实数据
  - 支持对重点关注股票重新分析
  - 提供动作变化 / 理由变化提醒
  - 收集股票结果反馈
  - 收集 skill 质量反馈
  - 让偏好学习影响排序与结果解释
  - 让历史记录页完整接入：按日期 / 按 skill / 按股票查看
- 达成标准：
  - 可以长期跟踪重点关注股票
  - 可以保留并比较多次重新分析结果
  - 反馈可以影响后续排序或模型权重
  - 历史记录形成完整闭环
- 前置依赖：
  - `Phase 4` 已生成真实 AI 结果与历史基础数据
  - 重点关注与历史记录页面已具备真实数据接入条件
- 非目标：
  - 不在本阶段重新定义前面阶段的数据底座与 skill 工作流
