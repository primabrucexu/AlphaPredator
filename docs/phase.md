# AlphaPredator 阶段目标清单

## 文档用途

- 本文件只维护 **阶段目标、阶段范围、关键任务、达成标准**，供 AI agent 和开发者理解“接下来要做什么”，和当前任务进度

## 维护规则

- 每个阶段都按固定模板维护，字段顺序保持一致。
- 只写该阶段的目标和计划，不写“已完成 / 未完成 / 进行中”。
- 如果阶段范围变化，优先更新：`本阶段范围`、`关键任务`、`达成标准`。
- 如果阶段之间存在依赖关系，写在 `前置依赖`，不要混写到 `关键任务` 中。

## 固定模板

以下每个阶段都按同一模板维护：

- 阶段目标：本阶段最终要实现什么
- 前置依赖：进入该阶段前必须具备的条件
- 本阶段范围：本阶段要覆盖的能力边界
- 关键任务：为达成目标需要完成的关键工作
- 达成标准：如何判断该阶段可以结束

## Phase 1：模仿常规股票行情软件

- 阶段目标：具备常规股票行情软件的核心功能，支持个股行情查看与搜索，满足日常选股分析的基础使用。
- 前置依赖：无。
- 本阶段范围：
  - A 股基础行情数据的采集、存储与查询。
  - 个股搜索、基础行情展示和常用技术指标展示。
  - 后端 API 与前端页面打通。
- 关键任务：
  - 建立稳定的行情数据导入流程（至少覆盖日线级别）。
  - 提供个股搜索与详情查询接口，并支持前端消费。
  - 实现常用技术指标计算与展示（如均线等基础指标）。
  - 完成基础数据存储方案落地（SQLite/DuckDB 的职责划分与使用）。
- 达成标准：
  - 用户可通过页面搜索股票并查看基础行情。
  - 系统可稳定展示日线行情与基础技术指标。
  - 行情数据可持续导入并可回查。

## Phase 2：短线情绪总览

- 阶段目标：建立韭研公社复盘数据的独立抓取 → 解析 → 存储 → 展示完整链路，支持用户按需获取并追踪热点板块动态。
- 前置依赖：完成 Phase 1 的基础行情数据与页面能力，完成韭研公社登录认证集成。
- 本阶段范围：
  - 复盘图片抓取任务（独立可控，类似市场数据初始化任务）。
  - 复盘图片解析与结构化（识别热点板块、题材、连板信息）。
  - 热点板块、题材、连板的数据模型与存储。
  - 热点数据的查询、对比与页面展示能力。
- 关键任务：
  - 实现独立的复盘图片抓取任务（支持指定日期范围、任务创建/启动/进度跟踪/重试）。
  - 建立复盘图片解析模块，自动提取热点板块、题材、连板等结构化信息。
  - 设计热点板块、题材、连板的数据模型，并完成数据库存储方案。
  - 建立热点数据查询能力（按日期、按板块维度查询）。
  - 打通前端页面，展示当日热点板块列表、每天热点板块变化情况与连板情况。
- 达成标准：
  - 用户能够在数据初始化页面创建复盘图片抓取任务，指定日期范围并启动。
  - 系统能够后台执行任务，自动解析复盘图片并提取热点信息。
  - 提取的热点板块、题材、连板等数据能够稳定存储到数据库。
  - 页面能够展示指定交易日的热点板块列表和多交易日的热点变化对比。
  - 页面能够展示股票连板情况并支持按条件查询。

## Phase 3：操作复盘

- 阶段目标：建立可持续的个人交易操作复盘体系，支持按次记录操作过程，并形成可回看、可总结的得失沉淀。
- 前置依赖：无。
- 本阶段范围：
  - 单次操作记录的结构化录入（标的、时间、动作、理由、仓位、结果）。
  - 复盘总结机制（预期与实际对比、得失归因、改进动作）。
  - 复盘记录的查询与回看能力（按日期、标的、策略维度筛选）。
- 关键任务：
  - 设计操作记录与复盘总结的数据结构，覆盖交易前、交易中、交易后关键信息。
  - 提供操作记录录入能力，支持快速新增、编辑、补充结果数据。
  - 建立标准化复盘模板，明确成功经验与失败教训的记录规范。
  - 提供复盘记录检索与列表展示能力，支持按条件回看历史操作。
- 达成标准：
  - 用户能够在每次操作后完成结构化记录，并保存到系统中。
  - 用户能够基于统一模板完成得失总结，并记录可执行的改进项。
  - 系统能够按日期、标的或策略维度查询历史复盘记录并稳定展示。

## Phase 4：AI学习用户选股模式

- 阶段目标：AI 能学习用户的选股模式，基于用户提供的历史选股数据与描述复刻其成功经验，并给出个性化建议。
- 前置依赖：完成 Phase 1 的行情底座与 Phase 2 的情绪数据沉淀。
- 本阶段范围：
  - 用户选股样本与行为数据接入。
  - 选股模式抽取、保存、回测与建议生成。
  - 用户反馈闭环。
- 关键任务：
  - 设计并实现用户选股数据导入与标注机制。
  - 建立模式抽取与特征表示流程，支持持久化保存。
  - 建立历史回测流程，评估模式有效性与稳定性。
  - 输出个性化选股建议，并支持用户反馈回流优化。
- 达成标准：
  - AI 能分析用户历史选股行为并提取选股模式，且可持久化保存。
  - AI 能在历史数据上对模式进行回测，验证有效性和稳定性。
  - AI 能基于用户模式和当前市场数据给出个性化选股建议。
  - 用户能够在页面上查看AI的选股建议，并且能够对AI的建议进行反馈，帮助AI进一步优化选股模式。

## Phase 5：AI自主选股

- 阶段目标：AI 将用户自然语言描述的选股思路转成可执行 skill，基于该 skill 自主选股，并通过多模型交叉评判提升结果可靠性。
- 前置依赖：完成 Phase 4 的用户模式学习与反馈机制。
- 本阶段范围：
  - 自然语言到选股 skill 的结构化编译与用户确认。
  - 基于 skill 的自主选股执行与解释输出。
  - 多模型对同一 skill/结果的交叉评判与一致性分析。
  - 基于反馈的 skill 迭代与历史验证。
- 关键任务：
  - 定义 skill 表达格式与执行约束，支持用户确认与版本管理。
  - 构建自主选股执行链路，输出候选、理由、风险与动作建议。
  - 接入多模型评审流程，对同一描述和结果进行交叉打分与分歧提示。
  - 建立 skill 调整后的回测机制，持续评估稳定性与收益风险特征。
- 达成标准：
  - AI能够将用户的自然语言描述转成可执行的 skill，并且能够让用户确认这个 skill。
  - AI能够基于这个 skill 进行自主选股，输出候选股票、理由、风险和动作建议。
  - 系统支持多个模型对同一份描述或结果进行交叉评判，并输出一致性/分歧结论。
  - 用户能够在页面上查看 AI 的选股结果，并能够反馈以帮助进一步调整 skill。
  - AI 能基于用户反馈调整 skill，并在后续选股中体现调整效果。
  - AI 可以根据提炼出来的 skill，检查其在历史数据上的表现，验证有效性和稳定性。
