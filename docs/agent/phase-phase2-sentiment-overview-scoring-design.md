# Phase 2 短线情绪页面设计（含情绪温度与板块影响公式）

## 背景

- 需求来源：用户要求先完成 `SentimentOverviewPage` 的整体设计文档，并重点明确：
  1. 情绪温度如何计算；
  2. 计算该公式需要哪些数据。
- 阶段上下文：当前处于 Phase 2（短线情绪总览）。
- 已确定交互方向：
  - 页面目标：同时回答 A（市场情绪强弱）+ B（主线板块识别）；
  - 布局：左右双栏（左 7 / 右 5）；
  - 联动：轻联动；
  - 分档：5 档；
  - 更新频率：收盘后更新。

## 目标

- 做什么：
  - 给出可解释、可扩展、可落地的情绪温度评分公式（0-100）。
  - 给出“某板块对当日情绪影响程度”的计算口径（贡献点数）。
  - 给出实现上述计算所需的数据清单（MVP / 增强版）。
- 不做什么：
  - 不在本次文档中直接修改数据库表。
  - 不在本次文档中承诺盘中实时评分。

## 页面信息架构（v1）

- 左栏（全市场视角，始终不切换）
  - 今日情绪结论卡：`分数 + 档位标签 + 较昨日变化 + 因子解释`
  - 近 N 日温度走势：用于识别拐点和连续性
- 右栏（主线识别）
  - 主线 Top 5（热度、持续天数、代表股数）
  - 选中主线趋势小图
- 底部明细区（默认展开）
  - 复盘图 + 补充明细（后续可扩展个股明细）
- 联动规则（轻联动）
  - 点击右栏主线后，不切换左栏主图，仅显示“贡献提示条”（例如：`机器人 +8，第1，连续3日入榜`）。

## 情绪温度评分公式（重点）

### 1) 总体结构

定义交易日 `t` 的原始情绪分数 `S_t`：

```text
S_t = wB * B_t + wM * M_t + wR * R_t + wL * L_t
```

- `B_t`：广度（Breadth）
- `M_t`：动量（Momentum）
- `R_t`：风险偏好（Risk Appetite）
- `L_t`：流动性（Liquidity）

初始权重（可调）：

```text
wB = 0.30, wM = 0.30, wR = 0.25, wL = 0.15
```

### 2) 指标标准化（鲁棒、抗极值）

对任意基础指标 `x_{i,t}`，先做鲁棒 Z 分数：

```text
z_{i,t} = (x_{i,t} - median_i,252) / (1.4826 * MAD_i,252)
```

再映射到 0-100：

```text
score_{i,t} = 50 + 50 * tanh(direction_i * z_{i,t} / 2)
```

- `direction_i = +1` 表示指标越大情绪越强（如涨停家数）
- `direction_i = -1` 表示指标越大情绪越弱（如炸板率）

每个维度分数由该维度内指标加权平均得到。

### 3) 平滑与切档

平滑分数（用于页面展示）：

```text
S*_t = 0.65 * S_t + 0.35 * S*_{t-1}
```

分档（5 档）：

- `0-19`：冰点
- `20-39`：弱修复
- `40-59`：震荡
- `60-79`：偏强
- `80-100`：亢奋

防抖规则：仅当跨档幅度超过 3 分时切档，降低边界抖动。

## 板块影响程度计算（重点）

### 1) 板块当日冲量

对板块 `s` 在交易日 `t` 定义冲量：

```text
Impulse_{s,t} =
  a1 * Heat_{s,t}
+ a2 * Continuation_{s,t}
+ a3 * Breadth_{s,t}
+ a4 * MoneyFlow_{s,t}
- a5 * Failure_{s,t}
```

含义：

- `Heat`：板块热度（如涨停家数）
- `Continuation`：延续性（如连续上榜天数、晋级率）
- `Breadth`：板块内部强度覆盖
- `MoneyFlow`：资金承接
- `Failure`：风险（炸板、回落）

### 2) 乘以市场暴露度

```text
RawImpact_{s,t} = Impulse_{s,t} * Exposure_{s,t}
```

- `Exposure_{s,t}`：板块在全市场中的权重（例如成交额占比）

### 3) 换算成“贡献点数”

令 `ΔS_t = S*_t - S*_{t-1}`，则：

```text
ImpactPts_{s,t} = ΔS_t * RawImpact_{s,t} / Σ_k |RawImpact_{k,t}|
```

解释口径：

- `ImpactPts > 0`：该板块拉升情绪
- `ImpactPts < 0`：该板块拖累情绪
- 建议单板块贡献封顶：`[-15, +15]`

## 公式所需数据清单（重点）

## A. 必需数据（MVP，先跑通）

### 全市场（日级）

- `trade_date`
- `rising_count`、`falling_count`
- `limit_up_count`、`limit_down_count`
- `highest_board_count`
- `total_turnover_billion`

### 个股事件（日级聚合来源）

- `stock_code`、`trade_date`
- 是否涨停/跌停
- 连板数（可由 `streak_text` 解析）
- 失败事件标记（至少支持炸板率计算）

### 板块（日级）

- `trade_date`、`sector_name`
- `heat_score`（板块热度）
- `max_board_count`
- `streak_days`（连续上榜天数）
- `sector_turnover_billion`（用于 `Exposure`）

### 评分结果（日级）

- `trade_date`
- `raw_score`（S_t）
- `smoothed_score`（S*_t）
- `score_label`
- `delta_score`

### 板块归因结果（日级）

- `trade_date`、`sector_name`
- `impact_points`
- `impact_rank`
- `confidence`

## B. 增强数据（V2，提升准确性）

- 盘中级触板/回封序列（提升炸板、承接强度口径）
- 板块净流入或更细资金流
- 高位股风险事件标签（天地板、大面等）
- 历史样本回测标签（用于权重再训练与分档校准）

## 实现建议与口径约束

- 频率：收盘后更新，不做盘中实时分。
- 解释优先：页面展示“结论 + 3 个主要原因”，隐藏复杂计算细节。
- 版本化：将参数（权重、阈值、平滑系数）配置化，并保存 `formula_version`。
- 稳定性：保留“上一交易日分数/标签/贡献”用于对比文案。

## 风险与缓解

- 风险 1：初始权重主观性较强。
  - 缓解：保留回测集，按历史样本评估分数与次日市场行为的一致性。
- 风险 2：炸板/资金流等字段缺失导致解释偏差。
  - 缓解：MVP 阶段允许低置信度输出，并在页面标注“估算”。
- 风险 3：边界抖动导致标签频繁跳变。
  - 缓解：EMA 平滑 + 切档滞回阈值。

## 验收标准

- 页面可稳定展示：`分数 + 5 档标签 + 较昨日变化`。
- 右栏 Top 5 每个板块可展示贡献点数并支持轻联动提示。
- 任意交易日可追溯：基础指标 -> 维度分 -> 总分 -> 板块贡献。
- 文档中的公式和数据字典可直接被后续实现任务引用。
