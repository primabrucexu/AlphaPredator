# F02：市场数据

## 背景

- 项目需要 A 股基础行情数据支撑个股搜索、行情展示、技术指标和热点复盘联动。
- 当前市场数据提供商为麦蕊智数，接口文档见 [mysj.yaml](../human/api-docs/mysj.yaml)。

## 目标

- 获取股票列表。
- 获取天级别交易数据，包括开盘价、收盘价、最高价、最低价、成交量等信息。
- 获取 5 分钟 K 线交易数据，支撑联动套利分析。
- 在数据初始化页面支持创建股票列表同步任务和天级别交易数据同步任务。
- 支持全量同步和增量同步。
- 支持涨跌停价格计算和股票所属市场判断。

## 不做什么

- 不在本文档中维护麦蕊智数完整 API 细节，接口细节以 [mysj.yaml](../human/api-docs/mysj.yaml) 为准。
- 不在本文档中定义或修改数据库表结构，数据模型以 [AlphaPredator.dbml](../human/data-model/AlphaPredator.dbml) 为准。

## 设计方案

- 数据来源：
  - 通过麦蕊智数接口抓取股票列表和天级别交易数据。
  - 麦蕊接口请求使用全局 paced limiter 进行统一流量控制；速率阈值由数据初始化页保存到 `data/config/mairui.json` 后生效，不允许一次性突发消耗一分钟额度。
  - 初始化任务实际拉取并发上限由后端根据当前速率自动推导；运行中根据最近请求的实际网络耗时自动升降并发窗口，用于尽量跑满配置速率且避免超过速率；写入 DuckDB 仍串行执行。
- 数据存储：
  - 股票列表保存在 SQLite 的 `stock_list` 表中。
  - 天级别交易数据保存在 DuckDB 的 `daily_price` 表中。
  - 5 分钟 K 线交易数据保存在 DuckDB 的 `5m_level_trade_data` 表中。
- 股票列表加载：
  - 在数据初始化页面创建股票列表同步任务。
  - 通过麦蕊智数接口获取股票列表并写入 SQLite。
- 天级别交易数据加载：
  - 在数据初始化页面创建天级别交易数据同步任务。
  - 遍历股票列表，按当前速率自动推导并发上限，并根据实际网络耗时动态调整并发窗口拉取历史数据，写入 DuckDB 时仍串行执行。
  - 全量同步：获取所有股票历史数据，适用于首次加载或数据严重过时的情况。
  - 增量同步：仅获取自上次同步以来有更新的股票数据。
  - 一键增量同步：根据最近一次成功的 `MARKET_DATA` 任务记录，自动从该任务 `end_date` 的下一天补齐到当前日期。
  - 数据初始化页展示最近一次成功行情同步的区间和完成时间，便于判断本地数据新鲜度。
  - 首页展示上一次行情同步区间和完成时间。
- 5 分钟 K 线数据加载：
  - 在数据初始化页面创建 `MARKET_DATA_5M` 同步任务。
  - 复用股票列表作为股票池，按当前速率自动推导并发上限，并根据实际网络耗时动态调整并发窗口拉取麦蕊 5 分钟历史 K 线。
  - 写入 DuckDB `5m_level_trade_data` 时仍串行执行，避免并发写入复杂度。
  - 单只股票写入使用 DataFrame 注册到 DuckDB 后 `INSERT SELECT` 批量入库，并使用 timestamp 边界删除目标区间旧数据。
- 涨跌停价格计算：
  - 主板普通 A 股：涨跌幅限制为 10%，涨停价为前收市价乘以 1.10，四舍五入至分。
  - 创业板 / 科创板：涨跌幅限制为 20%，涨停价为前收市价乘以 1.20，四舍五入至分。
  - 北交所：涨跌幅限制为 30%，涨停价为前收市价乘以 1.30，向下取整至分。
  - 上市后前 5 个交易日不设涨跌幅限制。
- 所属市场判断：
  - 主板：`60xxxx`、`00xxxx`。
  - 科创板：`688xxx`、`689xxx`。
  - 创业板：`300xxx`、`301xxx`。
  - 北交所：`920xxx`。

## 数据与接口依赖

- 麦蕊智数 API 文档：[mysj.yaml](../human/api-docs/mysj.yaml)。
- 数据模型：[AlphaPredator.dbml](../human/data-model/AlphaPredator.dbml)。

## 验收标准

- 用户可以在数据初始化页面创建股票列表同步任务。
- 用户可以在数据初始化页面创建天级别交易数据同步任务。
- 用户可以在数据初始化页面创建 5 分钟 K 线同步任务。
- 用户可以在数据初始化页面通过“一键增量更新”自动补齐最近成功行情任务之后的数据。
- 用户可以在数据初始化页面和首页看到上一次行情同步区间和完成时间。
- 股票列表可写入并查询。
- 日线行情数据可写入并查询。
- 5 分钟 K 线数据可写入 DuckDB `5m_level_trade_data`。
- 系统可以基于股票代码判断所属市场。
- 系统可以按市场规则计算涨跌停价格。

## 当前状态

- 本文档是市场数据需求的当前维护入口。
- 数据初始化页已支持基于最近成功 `MARKET_DATA` 任务的一键增量更新。
- 数据初始化页和首页已展示最近一次成功行情同步区间与完成时间。
- 行情初始化任务已改为任务内复用 DuckDB 连接；任一股票写入失败会直接中断任务并标记失败。
- 数据初始化页已支持创建 `MARKET_DATA_5M` 任务，按股票池拉取 5 分钟 K 并写入 `5m_level_trade_data`。
- `MARKET_DATA_5M` 写入已从 Python `executemany` 改为 DataFrame 注册 + DuckDB `INSERT SELECT`，并输出单股写入吞吐日志。
- 数据初始化页的麦蕊配置只需要维护请求速率；行情初始化任务并发上限由后端按速率自动推导，实际拉取窗口会根据当前请求耗时自动增加或降低。
- `MARKET_DATA` 日线导入按单股 DuckDB 最新交易日增量追加：首次导入全量拉取，后续只从该股最新日期的下一天拉取到任务结束日；若本地已覆盖任务结束日则跳过该股，不再删除已有日线数据。
- 后续需求变更应直接更新本文档。
