# 热点数据抓取设计

## 1. 数据来源

- 主要来源：韭研公社的复盘图片（包含热点板块、题材、连板等信息）

## 2. 数据抓取方式

通过韭研公社的API自动获取当日的涨停复盘信息和图片，API接口文档见：[jygs-api.yml](api-docs/jygs-api.yml)
。其中，api文档没指出的字段为不需要关注的信息字段

- 全量获取时，跟随个股每日行情初始化时获取。请求速率限制和个股行情相同，遵循个股行情的请求速率限制规则。
- 增量更新时：为了避免过于频繁的请求导致流控。我们仅在每天的 12:02 和 15:32 各抓取一次

## 3. 数据存储模型

参照 [AlphaPredator.dbml](data-model/AlphaPredator.dbml)

## 4. 数据使用

### 4.1 热点复盘页面展示

- 展示内容：当日的热点板块、题材、连板等信息，以及对应的复盘图片。
- 展示方式：在首页的热点复盘模块中展示，用户可以点击查看详情，查看当天的热点复盘信息和图片。
- 个股行情联动：在热点复盘模块中展示的个股，可以通过点击名称的方式跳转到详细行情页面，查看该个股的详细行情信息。
- 展示每个交易日的热点板块列表，并支持多交易日的热点变化对比。对比方式为：用折线图展示某个板块在多个交易日内，每天的涨停家数变化。并且允许通过点击对应图标的形式实现过滤
- 展示范围默认5个交易日，用户可以通过选择日期范围的方式调整展示的交易日范围。默认提供近3个交易日、近10个交易日、近20个交易日的快速选择按钮

### 4.2 AI学习用户选股偏好

- 通过分析用户提供的选股样本数据。分析其中包含的热点板块、题材、连板等信息，结合用户的选股样本数据，学习用户的选股偏好。

## 5. 备注

- 由于韭研公社需要登录后才能访问，我们需要在数据初始化页面中添加相应的跳转登录接口。待用户完成登录后保存相关登录凭据，每次获取前都检查该凭据是否可用，如果不可用则需要提示重新登录
- 韭研公社的直接访问地址为 https://www.jiuyangongshe.com/
