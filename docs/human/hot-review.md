# 热点数据抓取设计

## 1. 数据来源

- 主要来源：韭研公社的复盘图片（包含热点板块、题材、连板等信息）

## 2. 数据抓取方式

通过韭研公社的API自动获取当日的涨停复盘信息和图片，API接口文档见：[jygs-api.yml](jygs-api.yml)。其中，api文档没指出的字段为不需要关注的信息字段

- 全量获取时，跟随个股每日行情初始化时获取。请求速率限制和个股行情相同，遵循个股行情的请求速率限制规则。
- 增量更新时：为了避免过于频繁的请求导致流控。我们仅在每天的 12:02 和 15:32 各抓取一次

## 3. 数据存储

参照 [data-storage.md](data-storage.md) 文档中的

## 4. 数据使用

### 4.1 热点复盘页面展示

- 展示内容：当日的热点板块、题材、连板等信息，以及对应的复盘图片。
- 展示方式：在首页的热点复盘模块中展示，用户可以点击查看详情，查看当天的热点复盘信息和图片。

### 4.2 AI学习用户选股偏好

- 通过分析用户提供的选股样本数据。分析其中包含的热点板块、题材、连板等信息，结合用户的选股样本数据，学习用户的选股偏好。

## 5. 备注

由于韭研公社需要登录后才能访问，我们需要在数据初始化页面中添加相应的跳转登录接口。待用户完成登录后保存相关登录凭据，每次获取前都检查该凭据是否可用，如果不可用则需要提示重新登录
